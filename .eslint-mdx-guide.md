# ESLint Configuration for MDX Files

## Overview

This project enforces a **100% Component & MDX-driven architecture**. To maintain consistency and prevent the use of raw HTML elements in MDX files, we have configured ESLint with specific rules.

## Current Configuration

### ESLint Plugins Installed

- `eslint-plugin-react` - Provides React-specific linting rules including `forbid-elements`
- `eslint-plugin-mdx` - Provides MDX file support (currently not activated due to parser compatibility)

### MDX File Handling

Currently, **MDX files are excluded from automatic linting** (`**/*.mdx` in the ignores section) due to parser complexity with ESLint 9. However, the infrastructure is in place to enable it when needed.

### Code Review Guidelines for MDX Files

When reviewing MDX files, ensure developers follow these principles:

#### ✅ **DO** - Use Semantic Components

```mdx
import Card from '../../components/mdx/Card.astro';
import CardGrid from '../../components/mdx/CardGrid.astro';
import Callout from '../../components/mdx/Callout.tsx';

<CardGrid>
  <Card title="Example" href="/docs/example">
    Use components for structured content
  </Card>
</CardGrid>

<Callout type="info">
  Use Callout for important messages
</Callout>
```

#### ❌ **DON'T** - Use Raw HTML Elements

```mdx
<!-- Avoid this -->
<div className="card">
  <h3>Example</h3>
  <p>Content</p>
</div>

<section className="callout">
  Important message
</section>
```

### Available Components

For MDX content authors, use these semantic components:

- **`<Card>`** - For clickable cards with links
- **`<CardGrid>`** - Grid layout for cards
- **`<Callout>`** - For info boxes, warnings, tips
- **`<SearchHero>`** - Homepage search hero
- **`<DirectoryGrid>`** - Category directory grid

### Enabling ESLint for MDX (Future)

To enable the `react/forbid-elements` rule for MDX files:

1. Uncomment the rule in `eslint.config.js`:
   ```js
   'react/forbid-elements': ['warn', {
     forbid: [
       { element: 'div', message: 'Use semantic components like <Box> or <Card> instead of <div>' },
       { element: 'section', message: 'Use semantic components like <Card> or <CardGrid> instead of <section>' },
       { element: 'article', message: 'Use semantic components like <Card> instead of <article>' },
     ],
   }],
   ```

2. Remove `**/*.mdx` from the ignores section

3. Configure a compatible MDX parser when ESLint 9 support improves

## Manual Code Review Checklist

Until automatic linting is enabled for MDX files, reviewers should verify:

- [ ] No raw `<div>`, `<section>`, or `<article>` tags in MDX files
- [ ] All structured content uses semantic components (`<Card>`, `<CardGrid>`, etc.)
- [ ] Components are imported at the top of MDX files
- [ ] Styling is handled through component props, not inline classes
